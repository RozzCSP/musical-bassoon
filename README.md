## 🚀 Что было создано

### **1. Архитектура приложения**
- **Frontend**: HTML5 + Tailwind CSS + Vanilla JavaScript (никаких Framework зависимостей)
- **Backend**: PHP (совместимый с версией 7.4+) + PDO
- **Database**: SQLite или MySQL (по выбору)
- **API**: RESTful JSON API с аутентификацией через сессии

### **2. Функциональность платформы**

#### Для обычных пользователей:
✅ **Регистрация и авторизация** — создайте аккаунт и войдите  
✅ **Просмотр экскурсий** — каталог с фильтрацией по городу, цене и длительности  
✅ **Бронирование** — модальное окно с выбором даты и количества гостей  
✅ **Избранные экскурсии** — сохраняйте понравившиеся маршруты (работает с сердечком ❤️)  
✅ **Профиль пользователя** — просмотр бронирований и избранного  
✅ **Запрос на должность гида** — обычный пользователь может подать заявку, чтобы провести экскурсии  

#### Для гидов:
✅ **Управление экскурсиями** — гиды видят свои маршруты  
✅ **Профиль гида** — после одобрения администратором становятся гидом платформы  
✅ **Рейтинг** — отображение рейтинга каждого гида  

#### Для администратора:
✅ **Модерирование запросов** — рассмотрение заявок на роль гида  
✅ **Одобрение/отклонение** — управление доступом гидов  
✅ **Полный контроль** — система ролей (admin, guide, user)

## 📦 Технические компоненты

### **Frontend страницы:**
- **index.html** — главная страница с популярными экскурсиями
- **excursions.html** — каталог всех экскурсий с фильтрацией и добавлением в избранное
- **guides.html** — список всех гидов с поиском по имени и специализации
- **excursion.html** — страница деталей экскурсии с похожими маршрутами
- **profile.html** — личный кабинет с бронированиями, избранным и запросом на роль гида

### **REST API endpoints:**
```
POST   /api/register.php           — Регистрация нового пользователя
POST   /api/login.php              — Вход в систему
POST   /api/logout.php             — Выход из системы
GET    /api/get_excursions.php     — Получить список экскурсий (с фильтрацией)
GET    /api/get_guides.php         — Получить список гидов
GET    /api/get_excursion.php      — Получить информацию об одной экскурсии
GET    /api/get_profile.php        — Получить профиль пользователя
GET    /api/get_my_profile.php     — Получить расширенный профиль (с запросом гида)
POST   /api/book_excursion.php     — Забронировать экскурсию
POST   /api/add_favorite.php       — Добавить экскурсию в избранное
POST   /api/remove_favorite.php    — Удалить экскурсию из избранного
POST   /api/request_guide_role.php — Подать заявку на роль гида
GET    /api/get_guide_requests.php — Получить заявки гидов (только для админа)
POST   /api/approve_guide_request.php  — Одобрить/отклонить заявку гида (только для админа)
```

### **UX и интерфейс:**
✨ **Non-blocking Toast уведомления** — Пользователи видят информационные сообщения без блокировки интерфейса  
✨ **Модальное окно бронирования** — Динамически создаётся на каждой странице  
✨ **Адаптивный дизайн** — Работает идеально на мобильных, планшетах и десктопах  
✨ **Анимированные модали** — Гладкие переходы при открытии/закрытии  
✨ **Валидация форм** — Клиентская проверка перед отправкой на сервер

## 🗄️ База данных

### **Таблицы:**
```sql
users                 — Пользователи (role: admin, guide, user)
excursions            — Экскурсии (title, price, duration, city, guide_id, rating)
bookings              — Бронирования (user_id, excursion_id, dates, guests, status)
favorites             — Избранное (many-to-many между users и excursions)
guide_requests        — Заявки на роль гида (status: pending, approved, rejected)
```

### **Система ролей:**
- **admin** — администратор, может модерировать заявки гидов
- **guide** — гид, может проводить экскурсии
- **user** — обычный пользователь (может стать гидом через заявку)

## 🎨 Особенности UX

### **1. Toast-уведомления вместо alert()**
Все сообщения теперь показываются как неблокирующие уведомления в правом нижнем углу:
- ✅ Успешные действия (зелёные)
- ❌ Ошибки (красные)
- ℹ️ Информационные сообщения (серые)

### **2. Модальное бронирование**
- Динамически создаётся на списках экскурсий
- Работает и на странице деталей
- Закрывается по ESC, клику снаружи или кнопке
- Фокусировка на первый элемент формы

### **3. Добавление в избранное**
- Кнопка-сердечко ❤️ на каждой карточке экскурсии
- При добавлении — сердечко становится красным
- Одно нажатие добавляет/удаляет из избранного
- Данные синхронизируются с сервером в реальном времени

### **4. Фильтрация экскурсий**
- По городу, цене и длительности
- Данные обновляются на лету
- Результаты кэшируются локально для быстроты

### **Оптимизация изображений:**
Включен инструмент **Sharp** (Node.js) для автоматической обработки:
```bash
cd tour/tools/image-optimize
npm install
npm run optimize
```

Скрипт конвертирует изображения в WebP и создаёт:
- 📸 **Web версии** — max width 1600px (оптимизировано для веба)
- 🎬 **Миниатюры** — 400x300px для превью

**Исходные файлы**: `tour/images/originals/`  
**Результаты**: `tour/images/web/` и `tour/images/thumbnails/`
